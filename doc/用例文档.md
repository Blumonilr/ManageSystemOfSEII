| **参与者**   | 库存管理人员；目标是准确、简便地实现商品分类的增删改。                                                                                                                                                                                                                |                    |              |

| **前置条件** | 库存管理人员必须已经被识别和授权。                                                                                                                                                                                                                                    |                    |              |

| **前置条件** | 库存管理人员必须已经被识别和授权。                                                                                                               |                    |              |
| **后置条件** | 更新保存商品信息                                                                                                                                 |                    |              |
| **优先级**   | 高                                                                                                                                               |                    |              |
| **正常流程** | **1.0 增加商品**  <br> 1.库存管理人员选择“增加商品”功能               <br>      2.系统显示商品分类结构，其中叶节点的分类为可选择的增加商品的分类                <br>     3.库存管理人员选择要增加商品的叶分类                <br> 4.系统显示该叶分类下已有的商品和信息输入窗口（名称\*，型号\*，进价\*，零售价\*，库存数量），库存管理人员输入信息并确认（其中\*为必须输入的信息）<br> 5.系统自动生成商品其他属性（库存数量未输入的默认为0，自动生成编号，最近进价和最近零售价同进价、零售价），系统展示商品信息表               <br> 6.用户确认，系统返回增加商品界面，重复1\~6步直到商品增加完毕               <br>   7.用户选择完成，系统退出增加商品界面               <br><br>   **1.1 删除商品**             <br>  1.库存管理人员选择“删除商品”              <br> 2.系统显示商品分类结构和叶分类下的所有商品及商品编号               <br>      3.库存管理人员选择要删除的商品               <br>   4.系统显示商品的详细信息，询问“是否确认删除该商品？”              <br>   5.库存管理人员选择确认，系统删除该商品，更新商品信息并展示商品分类结构和叶分类下是所有商品及商品编号               <br> 6.重复1\~5步，直到删除商品完毕，库存管理人员选择完成，系统退出删除商品界面             <br><br>  **1.2 修改商品**             <br>   1.库存管理人员选择“修改商品”              <br> 2.系统显示商品分类结构和叶分类下的所有商品及商品编号               <br>  3.库存管理人员选择要修改的商品               <br>     4.系统显示可修改的商品信息表（可修改项为：名称、型号、进价、售价、库存数量）              <br> 5.库存管理人员修改信息并确认               <br>  6.系统更新并显示新的商品信息表               <br>  7.重复1\~7，直到修改完毕，库存管理人员选择完成，系统退出修改商品界面               <br><br> **1.3查询商品**             <br>  1.库存管理人员选择“查询商品”              <br> 2.系统展示商品分类结构，展示信息输入窗口                <br> 3.库存管理人员选择查询方式（关键字、编号等）、输入查询内容并确认               <br> 4.系统搜索符合条件的所有商品，展示商品列表（包括商品名称、编号）               <br>    5.重复1\~5，直到查询完毕               <br>   6.库存管理人员选择完成，系统退出商品查询界面                                                                            |                    |              |

| **特殊需求** | 1. 在修改商品和删除商品界面上要有查询商品的快捷键<br>    2. 商品的查询可以通过输入关键字、商品编号等进行模糊查找。                                                                                                           |                    |              ||
